# 结构化数据详解 - 给非技术人员

## 核心概念

**结构化数据 ≠ 页面显示内容**

- ❌ 不是给用户看的
- ❌ 不会改变页面外观
- ✅ 是给搜索引擎看的
- ✅ 改变的是Google搜索结果的展示

---

### HTML源代码（用户看不到）

```html
<!DOCTYPE html>
<html>
<head>
  <title>半导体材料研究讲座</title>
</head>
<body>
  <!-- 用户看到的内容 -->
  <h1>半导体材料研究讲座</h1>
  <p>演讲者：张三教授</p>
  <p>机构：清华大学</p>
  <p>时间：2024年1月15日 14:00</p>

  <!-- 结构化数据（隐藏的，用户看不到） -->
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "Event",
    "name": "半导体材料研究讲座",
    "startDate": "2024-01-15T14:00:00",
    "performer": {
      "@type": "Person",
      "name": "张三教授",
      "affiliation": {
        "@type": "Organization",
        "name": "清华大学"
      }
    }
  }
  </script>
</body>
</html>
```

**用户在浏览器里看不到<script>标签里的JSON数据**
**但Google的爬虫会读取它！**

## 类比说明

### 就像商品标签

```
商品本身：
  一瓶可乐（用户看到的）

商品标签（结构化数据）：
  - 品牌：可口可乐
  - 容量：500ml
  - 价格：3元
  - 生产日期：2024-01-01

用户看到的：
  一瓶可乐（外观不变）

超市系统看到的：
  标签上的所有信息（用于分类、搜索、展示）
```

**结构化数据就像商品标签，用户看不到，但系统能读取！**

## 你需要知道的

✅ **不会改变你的网站外观**
- 页面布局不变
- 颜色样式不变
- 内容位置不变

✅ **改变的是搜索结果**
- Google搜索结果更丰富
- 社交分享更美观
- 更容易被点击

✅ **用户感知不到**
- 用户访问你的网站，看到的和之前一样
- 但从Google搜索来的用户会更多（因为搜索结果更吸引人）

✅ **纯SEO优化**
- 不影响网站功能
- 不影响用户体验
- 只是让搜索引擎更好地理解你的内容
